# Contributing to ResQNet

Thank you for considering contributing to ResQNet!

## Development Setup

1. Ensure you have Flutter installed (version 3.38.4 or higher)
2. Clone the repository
3. Run `flutter pub get` to install dependencies
4. Create a new branch for your feature/fix

## Code Style

- Follow the official [Dart style guide](https://dart.dev/guides/language/effective-dart/style)
- Run `flutter format .` before committing
- Ensure `flutter analyze` passes without warnings

## Testing

- Write tests for new features
- Run `flutter test` before submitting PRs
- Ensure all tests pass

## Pull Request Process

1. Update documentation if needed
2. Add your changes to CHANGELOG.md
3. Ensure the app builds and runs without errors
4. Submit a PR with a clear description of changes

## Commit Messages

- Use clear and descriptive commit messages
- Start with a verb in present tense (Add, Fix, Update, etc.)
- Keep the first line under 50 characters
- Add detailed description if necessary

## Questions?

Feel free to open an issue for any questions or concerns.
